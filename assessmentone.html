Assessment One Deliverables

Assessment one overview

Deliverables

  • Website
    • Contains links to all documents related to project
    • Must be accessible at all times
  • Requirements
    • Elicitation and Negotiation
    • Research into requirements specification and presentation
    • User and System Requirements
  • Architecture
    • Diagrammatic representations of architecture of product
    • Tools used and reference specific langauges
    • Evolution of product with interim diagrams and CRC cards
    • Reference user requirements
  • Method Selection and Planning
    • Outline and justify software engineering methods
    • Identify and justify development and collaboration tools
    • Discuss alternative tools
    • Outline and explain team organisation
    • Give systematic plan for project with dates, priorities, and dependencies, using weekly snapshots
    • Evolution of plan
  • Risk Assessment and Mitigation
    • Risk management process
    • Risk register
  • Implementation
    • Documented code
    • code and executable JAR, including all external dependencies
    • 3rd-party libraries and assests and theu relevant licences
    • Suitability of licenses
    • Any requirements that haven't been fully implemented
